# SahajCode

**SahajCode** is a Nepali‑language programming language transpiler. It lets beginners
write programs using familiar Nepali keywords (in both Romanized Nepali and Devanagari
script) and transpiles them to readable **C99**, which is then compiled with `gcc` and run.

> Design goal (from the PRD): *Phonetic Intuition · Structural Familiarity · Minimal
> Syntax · Type Simplicity.* Only `int` and `string` exist in the MVP — no type
> declarations are needed; types are inferred.

## Installation

SahajCode requires **Python ≥ 3.10** and **gcc** (for the `run` command).

```bash
# Clone the repository
git clone https://github.com/Beelzebubo/SahajCode.git
cd SahajCode

# (Optional) install as a package so the `sahaj` command is available
pip install .
```

You can run programs in three equivalent ways:

```bash
python3 -m src.cli run hello.np      # recommended
python3 sahajcode.py run hello.np    # top-level wrapper
sahaj run hello.np                   # if installed via `pip install .`
```

## Quick Start

Create a file `hello.np`:

```sahajcode
# My first SahajCode program
bhana "Namaste Sathi"
```

Run it:

```bash
python3 -m src.cli run hello.np
# → Namaste Sathi
```

Under the hood SahajCode produces a C file (visible with `--keep-c`) and compiles + runs it:

```c
/* Auto-generated by SahajCode transpiler */
#include <stdio.h>
#include <string.h>

int main() {
    /* Line 2: bhana "Namaste Sathi" */
    printf("%s\n", "Namaste Sathi");
    return 0;
}
```

## Language Reference

### Keywords

| SahajCode | Devanagari | Meaning | C / Role |
|-----------|------------|---------|----------|
| `rakha`   | राख    | declare / keep | variable declaration |
| `bhana`   | भन     | say | `printf` (print) |
| `suna`    | सुन     | listen | `scanf` (input) |
| `yedi`    | यदि     | if | `if` |
| `bhane`   | भने     | then | marks the `if` body |
| `natra`   | नत्र     | else | `else` |
| `antya`   | अन्त्य   | end | closes a block `}` |
| `jaba`    | जब     | while | `while` |
| `samma`   | सम्म     | until | ends a `while` condition |
| `guma`    | गुमा    | loop / turn | `for` loop |
| `dekhi`   | देखि     | from | `for` start bound |
| `thik`    | ठीक     | true | `1` |
| `galat`   | गलत     | false | `0` |

Both Romanized and Devanagari forms are accepted interchangeably.

### Variables & Types

Types are **inferred** — you never write a type name.

```sahajcode
rakha x = 10          # int (inferred)
rakha nam = "Sunita"  # string (inferred)

x = 20                # reassignment (no `rakha`)
nam = "Ram"           # reassignment
```

- **MVP scope:** variables live in a single global scope for the whole program.
- Re-declaring an already-declared name (`rakha x = 5` then `rakha x = 6`) is a
  **semantic error (E004-style redeclaration)**. Use bare `=` to reassign.

### Input / Output

```sahajcode
bhana "Tapai ko nam?"   # print a string
suna nam                # read an integer into nam
bhana nam               # print it back

rakha msg = "Hello "
suna msg                # read a string into msg
bhana msg
```

- `bhana` prints with a trailing newline.
- `suna` reads an **int** by default, or a **string** when the target variable was
  declared as a string.

### Operators

| Category | Operators |
|----------|-----------|
| Arithmetic | `+  -  *  /  %` |
| Comparison | `==  !=  <  >  <=  >=` |
| Assignment | `=` |

- Unary minus is supported: `rakha y = -3`.
- Operator precedence follows standard arithmetic (`*` `/` `%` bind tighter than `+` `-`).
- String comparison (`==` / `!=` on strings) is compiled to `strcmp(...)`.

### Conditionals

```sahajcode
rakha score = 85

yedi score >= 90 bhane
    bhana "A+"
    bhana "Excellent!"
antya

yedi score >= 80 bhane
    bhana "A"
natra
    bhana "Below A"
antya
```

Structure: `yedi <condition> bhane` … `[natra …]` … `antya`.

### Loops

**While** — repeats while the condition holds:

```sahajcode
rakha count = 1

jaba count <= 5 samma
    bhana count
    count = count + 1
antya
```

**For** — inclusive range loop (`guma <var> = <start> dekhi <end> samma`):

```sahajcode
guma i = 1 dekhi 5 samma
    bhana i
antya
```

> Note: per the MVP grammar the `dekhi` bounds must be **integer literals**.

### Strings

```sahajcode
rakha greeting = "Namaste " + "Sahaj"   # concatenation
bhana greeting                          # → Namaste Sahaj

rakha age = 25
bhana "You are " + age + " years old"   # int auto-coerced to string
```

- `+` on strings concatenates; a string `+` an integer auto-converts the integer
  to its decimal string (PRD type-coercion rule).
- String equality uses `strcmp`, so `yedi nam == "Ram" bhane …` works correctly.

### Comments

```sahajcode
# This is a line comment — ignored by the parser
rakha x = 5   # trailing comments are fine too
```

## Error Handling

Every error is reported **bilingually** (Nepali + English) and mapped to the source
line. Error codes follow the PRD catalog:

| Code | Error | Example trigger |
|------|-------|-----------------|
| **E001** | Unterminated string | `bhana "oops` |
| **E002** | Missing `bhane` after `yedi` | `yedi x > 5` (no `bhane`) |
| **E003** | Undeclared variable | `bhana x` before `rakha x = …` |
| **E004** | Missing `antya` (end block) | unclosed `yedi` / `jaba` / `guma` |
| **E005** | Invalid character | `rakha x = @5` |
| **E006** | Division by zero | `rakha y = x / 0` (runtime) |

Example output:

```
[ERROR] Line 1: अमान्य अक्षर (Invalid character) '@'।
[ENGLISH] Line 1: Invalid character '@'.
          rakha x = @5
              ^
          Tip: संख्या 0-9 मात्र प्रयोग गर्नुहोस् / Tip: Use only digits 0-9.
```

### Process exit codes

| Code | Meaning |
|------|---------|
| 0 | Success |
| 1 | Lexical error (E001 / E005) |
| 2 | Syntax error (E002 / E004) |
| 3 | Semantic error (E003 / redeclaration) |
| 4 | C compilation failed |
| 5 | Runtime error (e.g. division by zero) |
| 6 | System error (e.g. `gcc` not found) |
| 7 | Input file not found |

## CLI Reference

```
python3 -m src.cli <command> [options]
```

### `run` — transpile, compile, and execute
```bash
python3 -m src.cli run program.np
```
| Option | Short | Default | Description |
|--------|-------|---------|-------------|
| `--keep-c` | `-k` | off | Keep the generated `.c` file |
| `--output` | `-o` | `a.out` | Name of the output binary |
| `--verbose` | `-v` | off | Print transpilation steps |

### `build` — transpile to C only
```bash
python3 -m src.cli build program.np          # writes program.c
python3 -m src.cli build program.np --show   # also print the C to stdout
python3 -m src.cli build program.np -o out.c # custom output path
```
| Option | Short | Description |
|--------|-------|-------------|
| `--output` | `-o` | Output C file path (default `<input>.c`) |
| `--show` | `-s` | Print generated C to stdout |
| `--verbose` | `-v` | Print transpilation steps |

### `init` — scaffold a project
```bash
python3 -m src.cli init myproject
```

### Global
```bash
python3 -m src.cli --version   # prints version (SahajCode v2.0)
```

## Project Structure

```
SahajCode/
├── src/                  # Transpiler source
│   ├── lexer.py          # Tokenizer (DFA, O(n))
│   ├── parser.py         # Recursive-descent parser → AST
│   ├── ast_nodes.py      # AST node definitions
│   ├── symbol_table.py   # Type tracking & semantic checks
│   ├── codegen.py        # AST → C99 visitor
│   ├── error_reporter.py # Bilingual error catalog (E001–E006)
│   └── cli.py            # `run` / `build` / `init` commands
├── tests/                # pytest suite (lexer, parser, codegen, CLI)
├── docs/                 # Guides & lesson plans (Nepali + English)
├── examples/             # Example programs
├── SahajCode_PRD_v2.0.md # Product Requirements Document
├── pyproject.toml        # Packaging metadata
└── README.md
```

## Development & Testing

```bash
pip install -r requirements.txt   # pytest, pytest-cov
python3 -m pytest tests/ -v       # run the full suite
```

The codebase is modular: the **lexer → parser → symbol table → code generator**
pipeline is swappable, and each stage is covered by unit and integration tests.

Code style & quality targets (from the PRD):
- Generated C compiles cleanly with `gcc -Wall -Wextra -std=c99`.
- Each generated C block is annotated with its source line
  (`/* Line N: <original code> */`).
